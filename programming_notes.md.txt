## ADaM — derivations and validation results

### ADSL
Built from DM, extended with HEIGHTBL/WEIGHTBL (from VS baseline), BMIBL/BMIBLGR1,
MMSETOT, DISCONFL, DCDECOD, DSRAEFL, DTHFL, EFFFL, COMP8FL/16FL/24FL.

Selected variables vs. reference: **exact match.**

Known gap: portfolio ADSL keeps some SDTM-derived vars not in reference ADSL; reference
has sponsor-specific vars not implemented here. Scope decision, not an error.

### ADQSADAS
Read from reference XPT (preserves ACITM01–14, ACTOT, LOCF, baseline/analysis flags).

Row count: **12,463 — exact match.**

PROC COMPARE flagged non-unique BY/ID vars even when using
`USUBJID + PARAMCD + AVISITN + DTYPE`. The same warning appeared in both portfolio
and reference datasets at the same ID pattern, confirming this is a QC key limitation
rather than a derivation defect.

### ADAE
Row count: **1,191 — exact match.** Serious TEAE pattern matches reference.

### ADVS
Direct VS→ADVS derivation, one row per SDTM VS record.

| | Rows |
|---|---|
| Portfolio ADVS | 29,643 |
| Reference ADVS | 32,139 |
| Gap | 2,496 |

Investigated in `validation/advs_deep_compare.sas`. Gap = derived analysis-visit
records (incl. End of Treatment) requiring visit-window logic not implemented here.
**Documented scope limitation, root cause confirmed.**

## SUPP domains

### SUPPAE
**Bug found and fixed:** initial build only wrote rows where QVAL='Y' (1,126 rows).
Reference comparison showed SUPP domains should carry one row per parent record
regardless of flag value. Corrected to write all 1,191 AE records, QVAL='Y'/'N'.

Final: 1,191 rows | QVAL='Y': 1,126 | QVAL='N': 65 — **matches reference.**

### SUPPDM
6 flags (SAFETY, ITT, EFFICACY, COMPLT8/16/24), whole-subject qualifiers.
Row count and QNAM distribution: **exact match.**

## Scope decisions (not bugs)

1. ADVS — simplified vs. reference visit-window/EOT logic (gap diagnosed, see above)
2. ADQSADAS — sourced from reference XPT rather than derived from QS
3. Some sponsor-specific reference ADSL vars not recreated
4. TFLs are portfolio-style, not full CSR shells
5. Validation is targeted, not full double programming
